Тестирование

Для первого билда
проводить Дымовое + Тестирование новой функциональности + Приемочное тестирование.

(см. Виды тестирования)

Для последующих билдов

  • если НЕ была добавлена новая функциональность, тогда проводить Дымовое + Дефект валидация + Минимальное приемочное;
  • если была добавлена новая функциональность, тогда:
    • вариант А: проводить Дымовое + Тестирование новой функциональности + Дефект валидация + Регрессионное тестирование + Приемочное тестирование;
    • вариант Б: проводить Тестирование новой функциональности + Дефект валидация + Регрессионное тестирование + Приемочное тестирование.

(см. Виды тестирования)

Очередность проверок

  • приоритет функциональности:
    • критический;
    • важный;
    • необязательный.
  • позитивное тестирование:
    • стандартный сценарий + корректные данные;
    • альтернативный сценарий + корректные данные.
  • негативное тестирование (обработка отказов):
    • стандартный сценарий + НЕкорректные данные;
    • альтернативный сценарий + НЕкорректные данные;
  • НЕстандартный сценарий + корректные данные (работа в двух вкладках браузера);
  • работа в НЕобычных условиях:
    • отключены картинки, JavaScript;
    • низкая скорость передачи данных в сети Интернет (нестабильность);
    • устаревшая версия браузера;
    • быстрое переключение (перетаскивание);
    • много кликов на один элемент.
