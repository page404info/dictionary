ГОСТ-34, ISO 12207, ISO 15288

Информационная Система – это сбор, обработка, хранение, передача и представление данных.
Цель ИС – выигрыш во времени, в точности, в снижении рисков.

Принципы SMART

  • конкретный - конечные результаты ясны и понятны;
  • измеримый - определены показатели достижения результата (в деньгах, в %, в количестве обращений);
  • достижимый - имеются ресурсы (знания, время, исполнители, деньги);
  • значимый - результат должен соответствовать потребностям;
  • своевременный - оптимальный срок достижения результата (декомпозировать долгосрочные цели, установить исполнителей и сроки).

Создание ИС

  • определить модель бизнес-процесса
    • установить объекты;
    • установить функции (цепочки действий);
    • установить участников (кто исполняет, кто руководит);
  • определить потоки данных (как происходит обмен информацией);
  • определить архитектуру системы (подсистемы, задачи, технологии);
  • определить интерфейсы (связь пользователя с системой).

Архитектура ИС

  • функциональная;
  • информационная;
  • системная;
  • программная;
  • данных.

Подсистемы ИС

  • информационное обеспечение - ввод, кодирование, обработка информации;
  • техническое обеспечение - технические средства на которых рахмещается ИС;
  • программное обеспечение - программы для обеспечения работы ИС;
  • организационное обеспечение - инструкции пользователя по подготовке и контролю информации;
  • математическое обеспечение - средства обработки информации;
  • лингвистическое обеспечение - средства общения с ИС;
  • правовое обеспечение - назначение ответственных лиц.
